Google TPU のスケーリング階層 — 全レイヤーが Google 独自技術

インターコネクト: ICI + OCS

  • ICI (Inter-Chip Interface): チップ間直接接続、3D Torus トポロジ
  • OCS (Optical Circuit Switch): MEMS ミラーで光パスを動的に切替
  • スイッチ数を 568台 → 48台に削減（OCS の効果）
  • NVIDIA NVSwitch に相当するが、完全に独自プロトコル

ソフトウェア: JAX + Pathways

  • JAX: 関数型 ML フレームワーク（PyTorch の代替）
  • XLA: コンパイラが自動で TPU コードに最適化
  • GSPMD: 自動並列化（手動設定不要）
  • Pathways: 分散ランタイム（ジョブスケジューリング）

インターコネクト: xGMI + Ethernet

  • Infinity Fabric / xGMI: 8 GPU フルメッシュ（128 GB/s/リンク）
  • Scale-out: 標準 Ethernet (RoCEv2)
  • 検証済み NIC: ConnectX-7、Broadcom Thor-2、AMD Pollara 400G
  • リファレンス設計: 128〜8,192 GPU クラスタ

  • ROCm 7.x: PyTorch 2.9、JAX 0.8.0 対応
  • HIP: CUDA コードの移植（cuda → hip 書き換え）
  • RCCL: NCCL 互換の集団通信ライブラリ
  • vLLM on ROCm が推論の推奨構成

性能ギャップの原因

MI300X + RoCEv2 が InfiniBand Non-SHARP の約半分にとどまる主因は RCCL のネットワーク最適化不足。ハードウェア（NIC）が同じ ConnectX-7 でも、ソフトウェアスタック（NCCL vs RCCL）の差が性能を決定する。Spectrum-X が IB に近い性能を達成できるのは NCCL との垂直統合が要因。

NVLink vs UALink — 標準化の競争

NVLink 6 (NVIDIA)

  • 帯域幅: 3,600 GB/s
  • プロプライエタリ（NVIDIA 制御）
  • NVLink Fusion: サードパーティへの限定的開放
  • 条件: NVIDIA の CPU/GPU との組み合わせが必須
  • パートナー: MediaTek, Marvell, AWS (Trn4), Fujitsu, Qualcomm

UALink 1.0 (オープン標準)

  • 帯域幅: 200 GB/s (x4 リンク) — NVLink 6 の 1/18
  • シグナリング: 200 GT/s (Ethernet L1 ベース)
  • AMD, Intel, Google, Meta, Microsoft が参加
  • UALoE: UALink over Ethernet でトンネリング
  • AMD MI400 シリーズで採用予定

帯域幅の差は圧倒的だが、オープン性に価値がある

NVLink 6 は UALink 1.0 の 18倍の帯域幅を持つが、NVIDIA エコシステムへのロックインを伴う。UALink はオープン標準として複数ベンダが実装可能であり、2.0 以降の帯域幅向上が期待される。AMD は Vulcano 800GbE NIC（2026年下期）で UALoE を実装予定。

集団通信ライブラリの群雄割拠

NCCL (NVIDIA)

業界標準。CUDA 専用。SHARP + NVLink と垂直統合。

RCCL (AMD)

NCCL fork、API 互換。SHARP 非対応。NCCL 比で性能劣位。

MSCCL++ (Microsoft)

NCCL 比 最大 2.8倍高速。RCCL 比 最大 3.8倍。Azure 最適化。

UCCL (UC Berkeley)

NCCL/RCCL のドロップイン代替。マルチベンダ NIC 対応（ConnectX, Broadcom, EFA）。

HetCCL (研究)

AMD + NVIDIA 混在トレーニング対応。同種比 98% スループット達成。

OneCCL (Intel)

Intel oneAPI エコシステム。Gaudi / Xeon 向け最適化。

08今後の展望

AI データセンター市場の構造変化と、各社の戦略的方向性。

NVLink Fusion の影響

NVIDIA がインターコネクトを開放することで、カスタム ASIC 市場を NVIDIA エコシステムに取り込む戦略。AWS Trainium4 が最初の大規模事例。競合チップですら NVIDIA のインフラに乗る時代に。

UALink の挑戦

AMD/Intel/Google/Meta が推進するオープン標準。現時点では NVLink の 1/18 の帯域幅だが、エコシステムの広がりと 2.0 以降の進化が鍵。Ethernet ベースの統一インフラへの道筋。

推論 ASIC の台頭

Cerebras + OpenAI、AWS Trainium の急速な成長、Groq の NVIDIA 統合。推論市場では汎用 GPU → 専用 ASIC へのシフトが加速。トレーニングは NVIDIA GPU が当面支配的。

NVIDIA のエコシステムロックインの構造

NVIDIA のデータセンター技術は、ハードウェアレベルでは多くが標準規格（PCIe、Ethernet、InfiniBand）に準拠しており他社製品と接続可能。しかし、ソフトウェアレイヤ（NCCL、CUDA、GPUDirect）での最適化が NVIDIA GPU に限定されていることで、実質的なエコシステムロックインを形成している。物理的な接続は可能でも、性能の最大化には NVIDIA GPU が必要という構造が、NVIDIA の競争優位の核心である。
